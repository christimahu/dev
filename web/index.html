Dev Environment

Containerized Development Environment

A seamless, cross-platform development environment that keeps your host system clean while providing a full suite of development tools. Develop with C++, Rust, Go, Python, and more with identical configuration across all your machines.

Quick Start

Get up and running in minutes with a complete development environment.

Version Controlled

Keep your configurations in source control. Never lose your setup again.

Installation

Prerequisites

  • Docker - For containerized environment
  • Python 3 - For setup and management scripts
  • Git - For version control

Installation Steps

# Step 1: Clone the repository
git clone https://github.com/christimahu/dev.git ~/dev

# Step 2: Run the setup script (host configuration)
cd ~/dev/scripts
python3 setup.py

# Step 3: Source your shell configuration
source ~/.bashrc  # or source ~/.zshrc for macOS/Zsh users

# Step 4: Build the Docker container
dev build

The installation has two main phases:

  • Host Setup (setup.py): Configures symlinks and sets up the dev command
  • Container Build (dev build): Builds the Docker container with development tools

Usage

Basic Usage

# Enter the development environment from any directory
cd ~/your-project
dev

# You'll now be in a shell inside the container
# with your project directory mounted

Common Commands

dev Enter a shell in the container
dev build Build the container image
dev status Show container status
dev exec [cmd] Execute a command in the container

Customization

The environment is designed to be easily customizable:

  • NeoVim Config: Edit ~/dev/config/init.lua
  • Shell Functions: Add to ~/dev/config/shell_functions
  • Docker Image: Modify ~/dev/Dockerfile

Commands Reference

Dev Command Options

dev Enter the development container shell
dev build Build the development container image
dev build --stage=full Rebuild the entire container from scratch
dev status Show status of development containers
dev exec [command] Execute a command in the container
dev stop Stop the running container
dev delete Delete the container
dev prune Clean up unused Docker resources
