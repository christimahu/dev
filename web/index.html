Why Use a Development Container?

Development environments can be complex to set up and maintain. Language versions, dependencies, and tools can conflict between projects. The Dev Environment solves this by providing a consistent, reproducible workspace that runs identically everywhere.

Installation

Prerequisites

  • Docker (Desktop for macOS/Windows, Engine for Linux)
  • Python 3.6+ (already installed on macOS, most Linux distros)
  • Git for version control
1

Clone the Repository

git clone https://github.com/christimahu/dev.git ~/dev
2

Run the Setup Script

cd ~/dev/scripts
python3 setup.py

This configures your host system with necessary symlinks and aliases.

3

Source Your Shell Configuration

# For Zsh (macOS default)
source ~/.zshrc

# For Bash
source ~/.bashrc
4

Build the Container

dev build

This builds the Docker container with all development tools. It may take a few minutes on first run.

5

Start Using Your Environment

cd ~/your-project
dev

Run the dev command from any directory to enter the development environment.

Usage

Common Commands

dev Enter a shell in the development container
dev build Build or rebuild the development container
dev status Show status of the development container
dev exec [cmd] Execute a command in the container
dev stop Stop the development container

When you exit the shell with exit, the container will automatically stop and remove itself.

Helper Functions

  • srv [port] [dir] - HTTP file server
  • tree [dir] - Enhanced directory listing
  • dockerrm - Remove all Docker containers
  • ghprune - Clean Git branches
  • tarfolder/untarfolder - Archive utilities

Customization

  • Edit config/init.lua for NeoVim settings
  • Add your own functions to config/shell_functions
  • Create custom.env for environment variables
  • Modify Dockerfile to add new tools
