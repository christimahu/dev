Features

Containerized Development

Keep your host system clean while having all development tools available. Each project gets its own isolated container.

Cross-Platform

Works on macOS, Linux, and Windows (with WSL). Same experience everywhere with identical tools and configurations.

Pre-installed Tools

C++, Rust, Go, Python, Node.js and more, pre-installed and configured. Language servers, linters, and formatters included.

Cloud Development

Google Cloud SDK with all major components pre-installed, including Cloud Storage, App Engine, Datastore, Pub/Sub emulators, and kubectl.

NeoVim Configuration

Modern NeoVim setup with LSP, Treesitter, completion, and more. Ready for serious development right out of the box.

Version Controlled Config

All configurations stored in Git. Sync between machines, track changes, and never lose your perfect setup again.

Helper Functions

Productivity utilities for various languages and tasks. Create new projects, serve files, manage git branches, and more.

Installation

Prerequisites

  • Docker - For containerized environment
  • Python 3 - For setup and management scripts
  • Git - For version control

Installation Steps


# Clone the repository
git clone https://github.com/christimahu/dev.git ~/dev

# Run the setup script
cd ~/dev/scripts
python3 setup.py

The setup script will:

  • Configure symlinks for NeoVim and shell configuration
  • Build the Docker container image
  • Compile included tools
  • Set up shell integration for the dev command

Post-Installation

After installation, restart your terminal or source your shell configuration: 


# For Bash
source ~/.bashrc

# For Zsh
source ~/.zshrc

You can now use the dev command from any directory: 


cd ~/your-project
dev

Usage

Basic Usage

The dev command is the main entry point to your development environment: 


# Enter the development environment from any directory
cd ~/your-project
dev

# You'll now be in a bash shell inside the container
# with your current directory mounted

Environment Features

  • Your home directory structure is preserved inside the container
  • NeoVim is pre-configured with the same settings as your host
  • Development tools for C++, Rust, Go, Python are ready to use
  • Helper functions are available for various tasks
  • Ports are forwarded for web development

Workflow Examples

C++ Development


# Use CMake to build a C++ project
cd ~/projects/cpp-project
dev

mkdir -p build && cd build
cmake ..
cmake --build .

Rust Development


# Build and run a Rust project
cd ~/projects/rust-project
dev

cargo build
cargo run

Serving Files


# Serve files from the current directory
cd ~/website
dev
srv

# Serve from a specific directory on a specific port
srv 8080 ./public

Customization

The environment is designed to be easily customizable:

  • NeoVim Configuration: Edit ~/dev/config/init.lua to customize your editor setup.
  • Shell Functions: Add your own functions to ~/dev/config/shell_functions.
  • Docker Image: Modify ~/dev/Dockerfile to add or remove tools, then rebuild with dev build.

Commands Reference

Dev Command

The dev command manages your development environment:

Command Description
dev Enter a shell in the development container
dev build Build the development container image
dev build --stage=full Rebuild the entire container from scratch
dev build --stage=user Rebuild from the user setup stage
dev build --stage=final Rebuild only the final stage
dev status Show status of development containers
dev exec [command] Execute a command in the development container
dev stop Stop the running container
dev delete Delete the container
dev logs View container logs
dev logs -f Follow log output
dev prune Clean up unused Docker resources
dev prune --all Remove all unused containers, networks, and images

Helper Functions

The environment includes numerous helper functions to boost your productivity:

File Utilities

  • srv [port] [dir] - Serve files via HTTP
  • tree [dir] - Enhanced tree command
  • tarfolder [dir] - Create a tar archive
  • untarfolder [file] - Extract a tar file

Git Utilities

  • ghprune - Prune git branches
  • ghdf - Git diff tool

GCP Utilities

  • gcpinfo - Show GCP configuration
  • gcpproject [id] - Switch GCP project
  • gcpbuckets - List GCS buckets

Terraform Shortcuts

  • tfinit - Initialize Terraform
  • tfplan - Plan Terraform changes
  • tfapply - Apply Terraform changes
